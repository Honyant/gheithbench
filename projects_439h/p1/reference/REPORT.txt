- What is the difference between concurrency and parallelism?
Concurrency is about dealing with multiple tasks at the same time, but not necessarily executing them at the same time
Parallelism is about executing tasks simultaneously, usually with multiple processors or cores.

- What are race conditions and how do they relate to concurrency and/or
  parallelism?
Race conditions are when a number of threads try to access or modify data at the same time, which results in undefined/erroneous behavior.
They relate to concurrency because concurrency is about multiple threads executing concurrently which can lead to race conditions. As for parallelism, due to the simultaneous execution of programs, race conditions are prone to happening.

- What is a critical section?
A critical section is part of program where resources are being shared, and it's important in these cases for only one thread to process the critical section at the same time to prevent race conditions.

- What is a barrier?
A mechanism that blocks groups of threads from proceeding until a certain number of threads have all reached the barrier. They are subsequently released to proceed onwards.

- What is a spinlock?
A lock where the thread trying to aquire the lock checks in a loop if the lock is available until it is.

- Read about the following x86 instructions and compare them:
     halt
     pause
     monitor/mwait

     halt stops the cpu until an interrupt happens, which helps improve power efficiency

     pause is helpful for spinlock loops by allowing other hyperthreads to use the cpu's resources, and improves power efficiency

     monitor/mwait, are two commands used together that are also helpful for power efficiency, monitor creates a range of memory addresses to watch, and mwait puts the cpu into a low power state until something in that memory range is written to.

- What is QEMU? Why is it useful?
 QEMU stands for quick emulator. it emulates a processor, capable of executing several different types of ISAs, and it also serves as a virtualizer.
 It's useful for running other OSs made for one machine on another machine.

- What is the MBR?
It is the Master Boot Record. It is the first sector on a bootable disk, and has info on the partitions of the disk and code to bootstrap and load in the OS.
